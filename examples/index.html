DesignScan

Any URL → design tokens. DesignScan renders a live site, observes what it actually paints, and emits a spec-compliant DESIGN.md (plus W3C tokens JSON and CSS variables) that an AI coding agent can read to match that site's look. Runs locally — no API key.

Quick start

One command, no API key — point it at any live site and get a DESIGN.md your coding agent can build with:

# one-time: install the headless browser
npx playwright install chromium

# any URL → DESIGN.md (Google Labs open format)
npx @designscan/extractor stripe.com --md --out stripe.DESIGN.md

# other formats, themes, and a visual proof sheet
npx @designscan/extractor vercel.com --format w3c --out vercel.tokens.json
npx @designscan/extractor linear.app --theme both --md --preview --out linear.DESIGN.md
--md DESIGN.md — YAML tokens + prose, passes the official linter --format w3c W3C Design Tokens JSON (Style Dictionary, Tokens Studio) --format css paste-ready CSS custom properties --format json the raw token profile

How it works

  1. Render & observe

    Headless Chromium loads the live page (hydration-aware) and reads the computed styles of every visible element — shadow DOM included. It even really hovers and presses the buttons to capture the true :hover / :active states, and probes the dark theme.

  2. Normalize into tokens

    Thousands of raw values are clustered into a clean profile: color roles (primary, background, text, borders…), the type scale, spacing and radius grids, shadows, container width and responsive breakpoints — preferring the site's own declared design system (--color-primary, --breakpoint-md) when the page really paints it.

  3. Emit a spec

    Out comes a spec-valid DESIGN.md, W3C tokens JSON, or CSS variables — plus a self-contained HTML preview that renders every token, and deterministic "notes for your coding agent" (contrast, hierarchy, font fallbacks).

Use it from your coding agent (MCP)

DesignScan ships an MCP server (listed on the official registry as io.github.ms-zaman/designscan). One tool — get_design_tokens(url, format?, theme?) — lets an agent pull live tokens mid-task: "restyle this app like stripe.com".

// e.g. Claude Code: .mcp.json
{
  "mcpServers": {
    "designscan": { "command": "npx", "args": ["-y", "@designscan/extractor", "mcp"] }
  }
}
